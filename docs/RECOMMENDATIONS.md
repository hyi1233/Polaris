# 可视化优先级推荐

> 基于用户价值、技术可行性和实现成本的综合评估

## 评估标准

| 维度 | 权重 | 说明 |
|------|------|------|
| **用户价值** | 40% | 解决真实痛点、提升效率的程度 |
| **技术可行性** | 30% | 数据获取难度、输出格式结构化程度 |
| **实现成本** | 20% | 开发工作量、依赖复杂度 |
| **使用频率** | 10% | 日常使用中需要查看的频率 |

---

## 🥇 Tier 1: 最优先实现（高价值 + 高可行）

### 1. MCP 服务器拓扑图

| 属性 | 评估 |
|------|------|
| 来源命令 | `claude mcp list` / `claude mcp get <name>` |
| 用户价值 | ⭐⭐⭐⭐⭐ 直接解决"MCP 服务器连没连上"的核心问题 |
| 技术可行性 | ⭐⭐⭐⭐⭐ 输出格式稳定，易于解析 |
| 实现成本 | ⭐⭐⭐⭐ 中低，单页拓扑图 |
| 使用频率 | ⭐⭐⭐⭐⭐ 每次调试 MCP 问题时都需要 |

**推荐理由**: MCP 是 Claude Code 的核心扩展机制，连接状态和健康检查是最高频的调试需求。拓扑图能直观展示所有服务器的连接状态、传输类型和认证状态，比阅读终端输出效率提升 10x。

**设计文档**: [mcp-prototype.md](visualization-design/mcp-prototype.md)

---

### 2. Auto-Mode 安全规则矩阵

| 属性 | 评估 |
|------|------|
| 来源命令 | `claude auto-mode config` / `claude auto-mode defaults` |
| 用户价值 | ⭐⭐⭐⭐⭐ 35+ 条安全规则文本阅读极其低效 |
| 技术可行性 | ⭐⭐⭐⭐⭐ 纯 JSON 输出，完美结构化 |
| 实现成本 | ⭐⭐⭐ 中等，规则分类引擎 + 矩阵渲染 |
| 使用频率 | ⭐⭐⭐ 安全审计和规则调优时 |

**推荐理由**: auto-mode 规则集包含 35+ 条安全规则，纯文本阅读和理解成本极高。矩阵图将规则按类别（Git/Credential/Infrastructure/...）和风险等级（允许/软拒绝/拒绝）二维展示，是信息密度最高的可视化形式。JSON 输出使得数据获取零成本。

**设计文档**: [auto-mode-prototype.md](visualization-design/auto-mode-prototype.md)

---

### 3. 插件生态管理面板

| 属性 | 评估 |
|------|------|
| 来源命令 | `claude plugin list` |
| 用户价值 | ⭐⭐⭐⭐⭐ 插件管理是日常高频操作 |
| 技术可行性 | ⭐⭐⭐⭐ 输出格式规律，解析简单 |
| 实现成本 | ⭐⭐⭐ 中等，需要 CRUD 交互逻辑 |
| 使用频率 | ⭐⭐⭐⭐⭐ 每次启用/禁用/安装插件时 |

**推荐理由**: 插件管理是 VS Code 成功的关键 UX 模式。Claude Code 已有 12+ 个插件，用户需要频繁启用/禁用/更新。面板式管理比命令行操作更直观，且可集成 marketplace 浏览功能。

**设计文档**: [plugin-prototype.md](visualization-design/plugin-prototype.md)

## 🥈 Tier 2: 次优先实现（中高价值 + 良好可行）

### 4. 工具权限矩阵编辑器

| 属性 | 评估 |
|------|------|
| 来源命令 | `--allowedTools` / `--disallowedTools` |
| 用户价值 | ⭐⭐⭐⭐ 权限配置是安全核心 |
| 技术可行性 | ⭐⭐⭐⭐ 纯配置，无运行时数据 |
| 实现成本 | ⭐⭐⭐ 需要模式匹配可视化 |
| 使用频率 | ⭐⭐⭐ 项目初始化和安全策略配置时 |

**推荐理由**: `--allowedTools "Bash(git *)"` 这种模式匹配语法对用户不友好。可视化编辑器可以将工具和 glob 模式以表格形式展示，支持点击添加/删除规则，最终生成正确的命令行参数。

**设计文档**: [cli-options-prototype.md](visualization-design/cli-options-prototype.md)

---

### 5. 会话管理时间线

| 属性 | 评估 |
|------|------|
| 来源命令 | `claude -r` / `claude -c` / `claude --from-pr` |
| 用户价值 | ⭐⭐⭐⭐ 多会话管理是高频需求 |
| 技术可行性 | ⭐⭐⭐ 需解析本地 session 存储 |
| 实现成本 | ⭐⭐⭐⭐ 需要文件系统访问和会话索引 |
| 使用频率 | ⭐⭐⭐⭐ 每次恢复会话时 |

**推荐理由**: 会话恢复（`-r`, `-c`, `--from-pr`）是日常高频操作。时间线视图展示所有会话的历史、分叉关系和上下文，比终端交互式选择器更高效。

**设计文档**: [cli-options-prototype.md](visualization-design/cli-options-prototype.md)

---

### 6. Doctor 健康检查仪表盘

| 属性 | 评估 |
|------|------|
| 来源命令 | `claude doctor` |
| 用户价值 | ⭐⭐⭐ 环境问题排查 |
| 技术可行性 | ⭐⭐ 终端 UI 输出，需 ANSI 解析 |
| 实现成本 | ⭐⭐⭐ 中等 |
| 使用频率 | ⭐⭐ 环境异常时 |

**推荐理由**: doctor 的交互式终端 UI 在非 TTY 环境不可用。结构化仪表盘可持久化检查结果，支持历史趋势追踪和团队环境对比。但 ANSI 解析增加了实现复杂度。

**设计文档**: [doctor-prototype.md](visualization-design/doctor-prototype.md)

## 🥉 Tier 3: 锦上添花（有则更好）

### 7. 模型选择器
- 交互式选择模型和 effort 级别
- 低实现成本，中等用户价值

### 8. 配置来源层级图
- 可视化 user/project/local 三级配置继承
- 有助于排查"配置不生效"问题

### 9. 认证状态卡片
- 集成到状态栏的认证信息展示
- 可合并到 auth 和 setup-token 管理

### 10. 版本管理模块
- 合并 install + update 的版本选择和更新提示
- 低频操作，但提升体验

## 技术栈建议

| 组件 | 推荐方案 | 理由 |
|------|---------|------|
| 渲染框架 | React + Ink (终端) 或 WebView (IDE) | Claude Code 已使用 Ink |
| 图表库 | Mermaid / D3.js | Mermaid 已在文档中大量使用 |
| 数据层 | CLI 输出解析器 | 无需 API，直接解析命令输出 |
| 状态管理 | Zustand / Jotai | 轻量，适合工具类应用 |
| 包管理 | monorepo (turborepo) | 多模块独立迭代 |
