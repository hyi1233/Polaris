## 快速开始

### 创建第一个任务

1. 打开 Polaris 应用，进入**调度器**页面
2. 点击**创建任务**按钮
3. 填写基本信息：
   - **任务名称**: 给任务起一个描述性的名称
   - **触发类型**: 选择触发方式（间隔、Cron、单次）
   - **触发值**: 设置触发时间
   - **引擎**: 选择 AI 引擎
4. 点击**保存**

### 启动调度器

1. 在调度器页面找到**调度器状态**区域
2. 点击**启动**按钮
3. 调度器会自动按设定的触发时间执行任务

## 任务模式

### 简单模式 (Simple)

**适用场景**: 一次性任务、简单定时提醒

简单模式直接使用 prompt 执行任务，每次执行都是独立的。

**配置项**:
- **提示词**: 发送给 AI 的任务描述

**示例**:
```
任务名称: 每小时服务检查
触发类型: 间隔执行
触发值: 1h
提示词: 检查以下服务的运行状态并报告异常：
- API 服务器
- 数据库连接
- 缓存服务
```

### 协议模式 (Protocol)

**适用场景**: 复杂开发任务、需要迭代的项目、长期维护任务

协议模式使用文档驱动工作流，AI 具有"记忆"能力，可以持续追踪进度。

**配置项**:
- **任务分类**: 选择任务类型
- **任务目标**: 描述整体目标
- **模板**: 选择预设模板或自定义
- **执行控制**: 最大执行次数、超时时间等

**特点**:
- 📝 持久记忆：AI 记住之前的工作
- 📋 任务队列：自动拆解和追踪子任务
- 🔄 迭代开发：支持渐进式完成
- 👤 用户补充：可随时调整方向

## 触发器配置

### 间隔执行 (Interval)

使用间隔表达式，格式：`数字 + 单位`

| 表达式 | 说明 |
|---|---|
| `30s` | 每 30 秒 |
| `5m` | 每 5 分钟 |
| `1h` | 每 1 小时 |
| `2h` | 每 2 小时 |
| `1d` | 每 1 天 |

**示例**:
- `5m` - 每 5 分钟执行一次
- `30m` - 每 30 分钟执行一次
- `1h` - 每小时执行一次

### Cron 表达式

使用标准 Cron 表达式，格式：`分 时 日 月 周`

| 表达式 | 说明 |
|---|---|
| `0 9 * * 1-5` | 工作日早上 9 点 |
| `30 14 * * *` | 每天下午 2:30 |
| `0 */2 * * *` | 每 2 小时 |
| `0 9 * * 1` | 每周一早上 9 点 |

**在线工具**: 可以使用在线 Cron 表达式生成器来帮助配置

### 单次执行

指定具体的执行时间，格式：ISO 时间戳

**示例**:
- `2024-03-16T14:00:00Z` - 在 2024 年 3 月 16 日 14:00 UTC 执行

## 协议模式详解

### 工作原理

协议模式创建一个任务文档结构：

```
.polaris/tasks/{taskId}/
├── protocol.md        # 任务协议文档
├── supplement.md      # 用户补充内容
└── memory/
    ├── index.md       # 记忆索引（进度追踪）
    └── tasks.md       # 任务队列
```

### 任务协议文档

协议文档定义任务的目标、规则和工作流程：

```markdown
# 任务协议

> 任务ID: abc123
> 创建时间: 2024-03-16 14:00:00
> 模板类型: development
> 版本: 1.0.0

---

## 任务目标

实现用户认证功能

## 工作区

```
/path/to/project
```

## 执行规则

### 1. 检查用户补充
- 读取用户补充文件
- 如有新内容，优先处理并归档

### 2. 推进主任务
- 读取记忆索引了解当前进度
- 选择下一个待办事项执行
- 完成后更新记忆

### 3. 记忆更新
- 新成果写入记忆文件
- 待办任务写入任务文件
```

### 用户补充机制

用户可以通过 `supplement.md` 文件临时调整任务方向：

```markdown
# 用户补充

> 用于临时调整任务方向或补充要求
> AI 处理后会清空内容，历史记录保存在 supplement-history/

---

请优先处理登录页面的 UI 实现，暂时搁置后端 API 开发。
```

### 记忆系统

**记忆索引 (memory/index.md)**:
- 记录当前状态和进度
- 已完成的成果列表
- 关键发现和决策

**任务队列 (memory/tasks.md)**:
- 待办任务列表
- 按优先级排序
- 每完成一项划掉一项

## 模板系统

### 内置模板

#### 功能开发 (dev-feature)

适用于持续开发新功能。

**参数**:
- `mission`: 任务目标（必填）

**使用场景**:
- 新功能开发
- Bug 修复
- 代码重构

#### 代码审查 (review-code)

适用于定期审查代码质量。

**参数**:
- `mission`: 审查范围（必填）

**使用场景**:
- 代码质量检查
- 安全审计
- 性能优化建议

#### 新闻搜索 (news-search)

适用于搜索和总结新闻。

**参数**:
- `keywords`: 搜索关键词（必填）
- `timeRange`: 时间范围（`1d` | `3d` | `1w` | `1m`）
- `sources`: 来源限制（可选）

**使用场景**:
- 行业新闻追踪
- 技术趋势监控
- 竞品动态跟踪

#### 服务监控 (monitor-service)

适用于监控服务状态。

**参数**:
- `service`: 服务名称（必填）
- `interval`: 检查间隔（`1m` | `5m` | `15m` | `30m`）
- `threshold`: 告警阈值（可选）

**使用场景**:
- 服务健康检查
- 性能监控
- 异常告警

### 创建自定义模板

1. 进入**模板管理**页面
2. 点击**创建模板**
3. 填写模板信息：
   - **名称**: 模板名称
   - **描述**: 模板用途说明
   - **分类**: 选择任务分类
   - **参数定义**: 定义模板参数
   - **协议配置**: 配置协议模板内容

**模板参数类型**:
- `text`: 单行文本
- `textarea`: 多行文本
- `select`: 下拉选择
- `number`: 数字
- `date`: 日期

**协议模板配置**:
- **任务目标模板**: 定义任务目标格式
- **执行规则**: 定义执行步骤
- **记忆规则**: 定义记忆格式
- **自定义区块**: 添加额外的文档区块

## 最佳实践

### 任务设计

1. **明确目标**: 任务描述应该清晰、具体
2. **合理间隔**: 根据任务重要性设置合适的触发间隔
3. **执行控制**: 设置最大执行次数和超时时间
4. **分类管理**: 使用分组和分类组织任务

### 协议模式使用

1. **任务拆解**: 将大任务拆分为小任务
2. **及时补充**: 通过用户补充调整方向
3. **定期回顾**: 检查记忆文件了解进度
4. **版本控制**: 将 `.polaris/tasks/` 纳入 Git 管理

### 性能优化

1. **避免高频触发**: 间隔不要低于 1 分钟
2. **设置超时**: 防止任务无限运行
3. **合理分组**: 将相关任务分组管理
4. **清理历史**: 定期清理已完成任务

## 常见问题

### Q: 任务没有按预期执行？

检查以下项目：
1. 调度器是否已启动
2. 任务是否已启用
3. 触发表达式是否正确
4. 引擎 ID 是否有效

### Q: 协议模式的文档在哪里？

协议文档位于项目根目录下的 `.polaris/tasks/{taskId}/` 目录中。每个任务都有一个唯一的 ID。

### Q: 如何修改协议任务的目标？

1. 进入任务详情页
2. 点击**编辑任务**
3. 修改任务目标字段
4. 保存后，协议文档会自动更新

或者直接编辑 `.polaris/tasks/{taskId}/protocol.md` 文件。

### Q: 如何查看任务执行历史？

目前任务执行历史保存在日志中。未来版本将提供更详细的执行历史查看功能。

### Q: 可以同时运行多个调度器吗？

不建议。调度器使用文件锁来防止冲突。如果启动多个实例，只有一个会持有锁并执行任务。

### Q: 如何备份任务配置？

任务配置保存在应用配置目录中：
- Windows: `%APPDATA%\polaris\scheduler\`
- macOS: `~/Library/Application Support/polaris/scheduler/`
- Linux: `~/.config/polaris/scheduler/`

协议文档保存在项目目录的 `.polaris/tasks/` 中。

### Q: 如何创建自定义模板？

1. 进入**调度器** → **模板管理**
2. 点击**创建模板**
3. 填写模板名称、描述和配置
4. 保存后即可在创建任务时选择

**注意**: 内置模板不能修改或删除，但可以创建基于内置模板的自定义模板。

### Q: 任务执行失败怎么办？

1. 检查任务日志了解失败原因
2. 验证 AI 引擎配置是否正确
3. 检查工作目录是否存在
4. 对于协议任务，检查文档结构是否完整
